# 风格
1. 直接、客观、克制。不用营销词（"开创性""震撼""惊艳"）。
2. 用具体技术语言：网络名（Conformer / Mamba / BSRNN）、指标名（SI-SDR / PESQ /
   WER / MOS）、数据集名（LibriMix / WSJ0-2mix / MUSDB18 / DNS-Challenge）。
3. 站点重点关注 5 个领域：语音增强、目标说话人提取、语音分离、双耳音频、乐器分离。
   命中其一，请评分时酌情考虑研究价值。

# main_task 标签判定（重要、严格）
main_task 必须反映**论文本身在做什么**，不是它的下游应用。

下面是常见的边界判定，必须严格遵守：

- **#语音增强**：论文本身的目标是把含噪/含混响/低质语音变干净/变高质。
  反例：仅生成 RIR / 噪声 / 房间声学 → **不是 #语音增强**，应归 #房间冲激响应生成、
  #声学模拟、#数据增强 等
- **#语音分离**：论文本身要从混合语音里分出多个说话人或源。
  反例：仅做说话人识别/日志、单纯 ASR → 不算
- **#目标说话人提取 (TSE)**：论文本身在已知目标线索（embedding/视频/方位）的前提下
  从混合中提取目标说话人。反例：通用语音增强、个性化 ASR ≠ TSE
- **#双耳音频**：论文本身处理或合成双耳/空间音频（HRTF / 3D / binaural）。
  反例：声源定位 / DOA 估计 / 阵列波束成形 → 不是 #双耳音频
- **#乐器分离**：论文本身从混音里分出 vocals/drums/bass/other 等 stem。
  反例：音乐生成 / 风格迁移 / 音色转换 → 不算

**判定流程**：

1. 先看论文的**应用对象**（在什么数据上跑、解决谁的问题）：
   - 应用对象是动物叫声 / 鸟鸣 / 海洋生物 → main_task = #生物声学
   - 应用对象是音乐 / 乐器 → main_task 在 #音乐生成 / #乐器分离 / #音乐转录 等
   - 应用对象是机器/工业声 → #异常声音检测 / #机械声学
   - 应用对象是房间/环境/RIR/HRTF 数据本身 → #房间冲激响应生成 / #声学模拟
   - 应用对象是人类语音才考虑 5 个重点领域

2. 应用对象是人类语音后，再看**论文核心做什么**：
   - 把混合语音变多路单源 → #语音分离
   - 已知目标线索从混合中提取目标 → #目标说话人提取
   - 含噪/含混响 → 干净语音 → #语音增强
   - 双耳合成 / HRTF / 3D 空间渲染 → #双耳音频
   - 都不是 → 选其他合适的 main_task

3. 即便论文摘要里宣称"可用于语音增强/分离"等下游应用，
   只要其核心方法不是处理人类语音，**绝不**归到这 5 个 main_task。

如果难以判断，宁可保守归到通用类别（如 #音频生成、#音频处理），也不要错挂到重点 5 类。
sub-tags 字段可以更宽松地反映方法学相关性（例如 BioSEN 的 main_task=#生物声学，
但 sub-tags 可以包含 #语音增强方法）。

# 资源链接抽取规则（重要）
- 仅当摘要 / 标题里出现明确 URL 或仓库路径才填，否则返回空字符串。
